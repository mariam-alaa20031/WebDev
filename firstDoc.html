    Q1: What does HTML stand for?

    Answer: HyperText Markup Language.

    Q2: What is HTML used for?

    Answer: Defining the structure of the webpage.

    Q3: What are HTML tags?

    Answer: These are the elements of the webpage that define its structure. These have 3 properties (first is a must):

    • Opening tag (example: <h1>)
    • Closing tag (example: </h1>)
    • Attributes (example: id in <h1 id='pageTitle'>)
    Note: each id value given to elements in an html page should be unique. 2 elements cannot have the same id.

    Q4: What are HTML element heading types?

    • <h2> : less important than h1

    • <h4> : less important than h1,h2,h3

    Q5: What are HTML element examples?

    • <header> : usually contains navbar and logo

    Q6: How to escape the rendering issues when using < or > or &?

    Answer: Using < , > or & usually would cause a problem in rendering html pages. Consider an example where you want to add a <section> as a content to an html element. How would the browser interpret it as a text and not an element in this case? To do this you have to learn how to escape them. Below is the trick:

    • < : '&' followed by 'lt' followed by a ';' with no space would give you the character itself
    • > : '&' followed by 'gt' followed by a ';' with no space would give you the character itself
    • & : '&' followed by 'amp' followed by a ';' with no space would give you the character itself

    Note: You do not add the single quotations. I'm just using them to specify the syntax. (example: by '&', I just mean a simple &)

    Q6: What are some encoding symbols to know?

    • © : '&' followed by 'copy' followed by a ';' with no space would give you the copyright symbol
    • " : '&' followed by 'quot' followed by a ';' with no space would give you the copyright symbol
    • None breaking space : This is used as a '&' followed by 'nbsp' followed by a ';' with no space, and it is used in cases where you want multiple sequential words in a paragraph for instance to always appear next to each other. Consider a case where a paragraph ending is 'the end'. If you squeeze the webpage from side, reducing its width, 'end' could appear on the beginning of the next line. This is the case where you could use an nbsp unicode. When used in between the 2 words, they will appear as 'the end' either on the end of the line before or on the start of the new line but always next to each other.

    Q7: Is choosing a certain page title using h1 tag important for SEO?

    Answer: Choosing the h1 content in html is believed to help a browser like Google and other browsers in search engine optimizations. Your page will be linked further in the top if it matches google's search prompt even more.

    Q8: What are the important 'a' tag attributes?

    • title: used in screen reader for blind people
    • href: used to specify the link of the page/section we want to redirect to or reach.
      • <a href='#id_value'> : id_value should be replaced by actual id of attribute value given to a certain element in the html page. This is used to jump to sections in a very long webpage for instance.
      • <a href='about.html'> : about.html in this case is another html page in the same folder directory of this webpage's html file. This is an example of a relative path.
      • <a href='https://www.google.com'> : the website in this case is an example of an absolute path.
    • target: <a target='_blank'> The _blank value is used to redirect to the href specified in a new tab.

    Q9: How are html elements displayed?

    Answer: HTML elements are either block elements or inline elements by default. Block elements would have the property of spanning the entire width of the screen. If you consider an example of placing 2 div elements as such inside the body tags, <div>This is the first Div</div><div>This is the second Div</div> , the second div would appear right below the first div. This is how block elements would display themselves in a browser. Some examples of these block elements include the div element, the h1,h2... elements and many other elements as well. Inline elements on the other hand can fit themselves on the same line. Consider a case of having a paragraph using p tag and then after the closing tag of the paragraph, you use a span element to write some few words after the paragraph. In this case, the content of the span element would appear in the same line as the last paragraph line. It wouldn't go on a whole new line by default because it's not a block element. Changing the display property of an element is possible (span's element display property could be changed to block instead of inline), but that's not the content of this section.

    Note: The browser displays the page elements according to the order you place them inside the body tags.
